## [C 语言扩展与跨编译器可移植性挑战](https://lemon.rip/w/6-c-extensions-compilers/) ⭐️ 7.0/10

lemon.rip 上的一篇文章讨论了 C 语言扩展在不同编译器间可移植性的持续挑战，指出了常见陷阱，如__attribute__的误用以及缺乏标准化检查。 这很重要，因为 C 仍然是系统编程的基础语言，可移植性问题阻碍了代码复用，并增加了跨平台和跨编译器的维护负担。 文章指出，许多项目使用 #if !(defined __GNUC__ || defined __clang__ || defined __TINYC__) 来忽略 __attribute__，但未检查 __attribute__ 是否已定义，导致与替代编译器出现兼容性问题。

hackernews · xngbuilds · May 25, 14:15 · [社区讨论](https://news.ycombinator.com/item?id=48267126)

**背景**: 像 __attribute__ 这样的 C 扩展是编译器特定的功能，增强了功能但破坏了可移植性。C 标准并未强制要求这些扩展，因此每个编译器以不同方式实现，导致针对多个编译器的开发者面临兼容性难题。

**社区讨论**: 评论者分享了经验：whizzter 指出以 Linux 为中心的代码常在 Windows 和 FreeBSD 上失败；WalterBright 描述了在 D 语言中实现 ImportC 的挑战；kscarlet 赞扬了 Common Lisp 生态系统的可移植性方法；fuhsnn 提供了 slimcc 的测试脚本和头文件技巧作为资源。

## [荷兰查获 800 台服务器，逮捕两名网络攻击协助者](https://krebsonsecurity.com/2026/05/netherlands-seizes-800-servers-arrests-2-for-aiding-cyberattacks/) ⭐️ 7.0/10

荷兰当局查获了 800 台服务器，并逮捕了两家托管公司的共同所有者，这些公司为俄罗斯提供基础设施，用于在欧盟内部进行网络攻击、影响力行动和虚假信息宣传活动。 此次行动打击了一个大型网络犯罪协助网络，并凸显了荷兰作为防弹托管中心的角色，向犯罪分子发出强烈信号：此类基础设施将受到打击。 被捕者此前曾在 2025 年 KrebsOnSecurity 的调查中被报道，查获的服务器对多个网络犯罪团伙的运营能力构成了重大打击。

hackernews · Krebs on Security · May 25, 13:56 · [社区讨论](https://news.ycombinator.com/item?id=48266906)

**背景**: 防弹托管是指无视非法活动投诉的服务，常被网络犯罪分子用于托管恶意软件、命令与控制服务器以及虚假信息网站。荷兰因其强大的互联网基础设施和历史上宽松的执法而成为此类托管的重要地点。

## [GerbDiff：PCB 设计文件可视化差异对比工具](https://www.gerbdiff.com/) ⭐️ 7.0/10

GerbDiff 是一款新的桌面应用程序，可逐层对比 PCB Gerber 文件的视觉差异，帮助设计者在制造前发现代价高昂的错误。 该工具解决了 PCB 设计中的一个关键痛点——未检测到的变更可能导致昂贵的制造错误，而其本地处理方式确保了设计 IP 的安全。 GerbDiff 拥有自主研发的 Gerber 解析器以确保可靠性，支持自动层检测，可将差异报告导出为 PNG 或 HTML 格式，并提供免费的 GitHub Action 用于自动集成到 PR 中。

rss · Hacker News Show HN · May 25, 21:00

**背景**: Gerber 文件是电子行业 PCB 设计数据传输的标准格式。设计者经常需要比较不同版本，以确保没有意外变更被发送到制造环节，但现有工具功能有限。GerbDiff 通过专用的视觉差异对比工具填补了这一空白。
